# Dashboard Overview Consistency Plan

## Objective
Create a consistent layout and structure for the overview pages of Business, Professional, and User dashboards, ensuring uniformity in design, components, and user experience while adapting to each role's specific data and functionality.

## Current State Analysis

### Business Dashboard Overview
- **Layout**: Header + Grid (2/3 main + 1/3 sidebar)
- **Main Content**: 4 stats cards, ProfileViewsChart, Product Snapshot card, Recent Inquiries card
- **Sidebar**: BusinessProfilePreview, AccountProgress, AccountSummary, QuickActions
- **Styling**: rounded-3xl cards, border-gray-300, shadow-none, hover:shadow-lg

### Professional Dashboard Overview
- **Layout**: Header + Grid (2/3 main + 1/3 sidebar)
- **Main Content**: 4 stats cards, ProfileViewsChart, MyServices card, MyProjects card, RecentEnquiries card
- **Sidebar**: ProfilePreview, AccountProgress, AccountSummary, QuickActions
- **Styling**: Same as business

### User Dashboard Overview
- **Layout**: Header + Stats grid + Content grid + Getting Started (no sidebar)
- **Main Content**: 4 stats cards, Recent Activity card, Quick Actions card, Getting Started section
- **No Sidebar**: Missing profile preview, progress, summary, actions
- **Styling**: rounded-lg cards, shadow-sm, border-gray-200

## Issues Identified

1. **Layout Inconsistency**: User overview lacks the sidebar structure used by Business and Professional
2. **Styling Inconsistency**: User uses different card styling (rounded-lg vs rounded-3xl, shadow-sm vs shadow-none)
3. **Missing Components**: User overview missing chart, profile preview, account progress/summary, and quick actions
4. **Content Structure**: User has "Getting Started" section not present in others
5. **Stats Adaptation**: User stats need to be adapted to 4 consistent cards

## Implementation Plan

### Phase 1: Create Missing User Components

1. **Create UserProfilePreview Component**
   - Location: `src/app/dashboard/user/components/UserProfilePreview.tsx`
   - Purpose: Display user profile information (name, avatar, member since, etc.)
   - Style: Match BusinessProfilePreview and ProfilePreview styling

2. **Create UserAccountProgress Component**
   - Location: `src/app/dashboard/user/components/UserAccountProgress.tsx`
   - Purpose: Show profile completion progress
   - Adapt from professional AccountProgress
   - Items: Profile info, avatar, bio, preferences, etc.

3. **Create UserAccountSummary Component**
   - Location: `src/app/dashboard/user/components/UserAccountSummary.tsx`
   - Purpose: Display account info (member since, plan, status)
   - Adapt from professional AccountSummary

4. **Create UserQuickActions Component**
   - Location: `src/app/dashboard/user/components/UserQuickActions.tsx`
   - Purpose: Quick action buttons (browse businesses, find professionals, update profile)
   - Adapt from professional QuickActions
   - Actions: Browse Businesses, Find Professionals, Update Profile, View Inquiries

5. **Create UserActivityChart Component**
   - Location: `src/app/dashboard/user/components/UserActivityChart.tsx`
   - Purpose: Display user activity/inquiry trends
   - Adapt from ProfileViewsChart or create simple placeholder
   - Show inquiry activity over time

### Phase 5: Testing and Validation
